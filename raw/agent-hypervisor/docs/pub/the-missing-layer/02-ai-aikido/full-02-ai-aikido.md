# AI Aikido: The Pattern Every Developer Uses Daily — Applied to Agent Security

*Part 2 of "The Missing Layer" — a series on architectural security for AI agents*

*Personal analysis — does not represent Radware's position*

---

The previous article established a pattern: every AI defense tested by researchers from OpenAI, Anthropic, and Google DeepMind broke under adaptive attacks. The root cause is architectural — agents process trusted and untrusted data in the same cognitive space, and every defense operates after the agent has already perceived dangerous input.

I also described the honest weakness of the alternative: if you virtualize the agent's perception of reality — replacing raw inputs with structured, trust-tagged semantic events — someone must still parse raw reality at the boundary. Parsing requires intelligence. Intelligence is stochastic. The deterministic architecture needs a non-deterministic component at its edge.

This paradox has an elegant resolution. And the resolution is a pattern the entire software industry already practices daily.

## The Pattern Hidden in Plain Sight

A developer asks GitHub Copilot to generate a function. Copilot produces code. The developer reviews it, commits it, and the function executes deterministically in production. A stochastic process produced a deterministic artifact.

ChatGPT generates a SQL query. The query executes against a database engine. Same input, same output, always. Claude generates a Terraform configuration. The configuration provisions infrastructure identically every time.

Every time an LLM generates code that is then compiled and executed, a stochastic process has produced a deterministic artifact. The LLM participates in the generation phase. The generated artifact executes without it.

The boundary between stochastic and deterministic does not run between systems. It runs between **phases**. Design-time is stochastic. Runtime is deterministic.

The entire industry practices this pattern. Nobody has named it. And nobody has applied it to the domain where it's most critically needed: agent security.

## AI Aikido

I call this principle **AI Aikido**: using the LLM's own capability to build the deterministic structure in which agents safely operate. The stochastic system constructs the deterministic system. Intelligence designs the physics; it does not govern the physics at runtime.

Applied to agent security:

Where Copilot generates **application code** → Agent Hypervisor generates **security parsers**. An LLM analyzes a corpus of real inputs — emails, documents, MCP schemas, API payloads — and generates regular expressions for known attack patterns, PEG grammars for structured input validation, JSON Schema validators for tool interfaces, canonicalization rules for Unicode normalization. Each artifact is deterministic, testable, verifiable. The LLM does not parse at runtime — code generated by the LLM parses at runtime.

Where Cursor generates **modules** → Agent Hypervisor generates **World Manifests**. Given a description of a business process, an LLM generates the set of permitted actions and their schemas, trust relationships between input channels, capability presets per trust level, taint propagation rules, escalation conditions. A human reviews, modifies, and commits. The manifest becomes the constitution of the agent's world.

Where ChatGPT generates **SQL** → Agent Hypervisor generates **taint propagation rules**. An LLM analyzes data flow and proposes: "If source is email and transformation is summarize, taint is preserved." "If transformation is count_words, taint is cleared." A human approves each rule. The rule becomes a deterministic physics law.

The same LLM (or a different one) then attacks the generated artifacts — generates adversarial inputs designed to bypass canonicalization, probes edge cases in grammar definitions, crafts semantic ambiguity attacks. The cycle: **generate → attack → patch → re-attack** — all at design-time. The parsers that survive are deployed.

## The Industry Is Already Converging on This

Look at the patterns emerging across unrelated domains:

**SOC bounded autonomy.** Tier-1 SOC work is becoming code. AI agents handle triage and enrichment automatically, but humans approve containment actions when severity is high. CrowdStrike documents breakout times of 51 seconds — manual triage can't compete. The response: let AI operate freely within defined boundaries, escalate to humans at explicitly defined thresholds. This is design-time human judgment amortized across thousands of runtime decisions.

**Voice AI modular architecture.** The enterprise voice AI market split into three architectures: native speech-to-speech (fast but opaque), legacy chained pipelines (auditable but slow), and unified modular (both). The winning architecture retains a text layer between transcription and synthesis — enabling PII redaction, memory injection, and compliance scanning at the boundary. Native S2S models are "black boxes." Modular architectures have intervention points. The parallel to agent security is direct: a layer between raw input and agent perception where deterministic policy can operate.

**Cloudflare Moltworker.** When OpenClaw hit 180,000 GitHub stars and researchers found thousands of exposed instances, Cloudflare released Moltworker: ephemeral containers that isolate the agent, encrypted storage for persistent state, Zero Trust authentication on the admin interface. The agent runs in a sandboxed world that dies when the task ends. A compromised agent gets trapped in a temporary container with zero access to the local network. This is primitive world definition — the agent's universe is the container, and the container is deterministic.

**Meta's Rule of Two.** "Guardrails must live outside the LLM. File-type firewalls, human approvals, and kill switches for tool calls cannot depend on model behavior alone." Meta is saying what AI Aikido formalizes: security logic must be deterministic and external to the model. The model's stochastic nature is exactly why enforcement must be elsewhere.

None of these projects use the same terminology. All of them implement the same pattern: define boundaries at design-time, enforce deterministically at runtime, keep intelligence out of the critical security path.

## The World Manifest Compiler

AI Aikido resolves the paradox from the previous article, but it needs a formal structure. That structure is the **World Manifest Compiler**.

The compilation pipeline:

```
Human intent + LLM semantic modeling
              ↓
     World Manifest (reviewed & committed)
              ↓
     Compilation phase
              ↓
     Deterministic runtime artifacts
              ↓
     Runtime enforcement (LLM-free)
```

The World Manifest is a formal document defining everything that exists in the agent's universe: the complete set of actions the agent can propose (Action Ontology), trust channels and their levels (Trust Model), which capabilities are available at which trust levels (Capability Matrix), how contamination spreads through data transformations (Taint Propagation Rules), explicit boundaries where deterministic decisions escalate to human review (Escalation Conditions), and how origin metadata propagates (Provenance Schema).

The compiler transforms this into executable runtime artifacts: policy lookup tables, JSON Schema validators, taint propagation state machines, capability resolution engines. **No LLM survives this phase.** The output is pure deterministic code — lookup tables, state machines, validators. Every compiled artifact is unit-testable. Every decision is reproducible.

At runtime:

1. Raw input arrives at the virtualization boundary
2. Compiled canonicalizer strips known attack patterns
3. Trust assignment table maps source → trust level
4. Taint propagation matrix computes contamination status
5. Capability lookup determines available actions
6. Agent proposes an intent (structured JSON only)
7. Deterministic policy engine evaluates: allow | deny | require_approval | simulate
8. Decision is logged with full provenance for audit

Same manifest + same input = same decision. Always.

## What This Solves — and What It Doesn't

**Coverage completeness is finite.** The World Manifest covers what was anticipated at design-time. Novel attack patterns require redesign and recompilation. But this is now a manageable engineering problem — parsers can be iterated, adversarial testing automated, coverage expanded incrementally.

**Semantic ambiguity is resolved by policy, not eliminated.** "Send the report to Alex" remains ambiguous. AI Aikido generates the policy rule, but the correctness of that rule is a human judgment.

**Adaptation is not instantaneous.** New attack → redesign → recompile → redeploy. Faster than manual authoring with AI Aikido, but a latency window exists. The gap is finite and measurable — unlike the infinite gap of hoping a runtime probabilistic filter catches an unknown pattern.

The honest claim: Agent Hypervisor turns security from an unbounded probabilistic problem into a bounded deterministic engineering problem. The bounds are explicit, measurable, and improvable through iteration. This is not the same as solving security. It is making security tractable.
